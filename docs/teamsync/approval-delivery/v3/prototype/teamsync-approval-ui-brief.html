TeamSync / 審核模組 UI 初版交付整理

審核模組 UI 調整說明

本文件整理目前已完成的五個介面原型，內容以 PM wireframe 與開發文件為基礎，再依照 TS 實際使用情境調整。

頁面整理

每頁包含：設計重點、主要調整、待優化與原型入口

1. 審核流程設定頁

將原本偏系統設定導向的流程頁，調整成可視化但低門檻的線性簽核編排工具。

開啟原型

設計重點

參考工作流形式，但限制為線性單向流程 保留可視化與排序能力，避免過度技術化。
關卡設定直接整合在節點內 不再使用複雜節點工具箱，降低視線切換。
新增關卡後預設可直接編輯名稱 減少額外點擊，順著建立流程心智操作。

相較 Wireframe 的主要調整

新增動態審核對象 支援直屬主管、職務代理人等依申請人帶入的角色。
補上找不到對應人時的處理 如不可送出、改送人資，避免流程卡住。
完成後可直接套用到表格 縮短設定完流程還要跳回表格的路徑。
增加退回規則與套用位置查看 可設定退回上一關或申請人，也可查看目前套用在哪些表格。

待優化

新增審核對象的彈窗 固定人員、群組標籤、動態角色的選擇方式仍可再細修。
建立群組標籤 可再串接至系統管理的群組標籤頁，讓流程中能快速使用既有人員群組。

2. 審核中心頁面

整理跨聊天室的待審與送審資訊，讓使用者先看得到要處理什麼、進度走到哪裡。

開啟原型

設計重點

以整頁工作台呈現，不綁定單一聊天室 符合審核資料來源跨聊天室、跨表格的特性。
高權限與一般使用者採不同視角 高權限可看待我簽核 / 我送出的；一般使用者只看我送出的。
詳細資訊依情境切換重點 待我簽核看審核動作，我送出的看進度、退回原因與重送。

相較 Wireframe 的主要調整

新增子表格明細 用途情境：差旅費申請一筆審核單內會有多項明細
新增右側資訊狀態 讓列表狀態更一致，並依請假 / 報帳等不同單據顯示不同明細。
加入來源跳轉 可前往來源聊天室或表格，方便追查上下文。

待優化

一般使用者精簡版 可再做一版更單純的「我送出的」視角，作為低權限預設頁。

3. 新建表格 Step 3：權限與啟用審核

把原本分散的權限與簽核設定合併成同一步，維持新建表格流程的簡潔度。

開啟原型

設計重點

整合權限與審核 避免五步驟過長，也減少畫面過擠。
審核設定可跳過 若尚未建立流程模板，不會卡在新建表格流程中。
先選既有流程，不在此頁承擔完整建流程 避免同一頁同時處理權限與流程編輯，負擔過重。

待優化

無流程時的引導文案 可再確認是要導去流程設定頁，還是保留更明確的下一步提示。

4. 表格設定的審核 Tab

讓已建立完成的表格，後續仍可在設定中獨立開關審核與更換流程。

開啟原型

設計重點

將審核獨立成設定頁籤 方便後續維護，不需重新走一遍建立表格流程。
保留啟用開關與流程切換 讓管理者可快速查看目前表格套用的是哪條流程。

待優化

與未來流程版本管理的銜接 若流程已有資料執行中，表格設定頁的提示與限制還可再補強。

5. 系統管理：帳號與群組標籤

整合帳號審核關係與群組標籤，集中管理直屬主管、職務代理人及可重複套用的人員群組。

開啟原型

設計重點

主畫面改為管理表格 第一眼可看到成員數、已套用位置與狀態，不必逐一點入確認。
帳號管理整合審核關係 在同一頁設定直屬主管與職務代理人，供動態審核角色直接帶入。
右側 Drawer 快速編輯成員 保留 PM 版點開後直接新增/移除成員的操作優點。
新增群組流程保持簡單 建立時只需先設定名稱與成員，降低管理者建立成本。

相較 Wireframe 的主要調整

補上套用位置與刪除防呆 避免修改或刪除正在被簽核流程、權限或通知使用的群組。
降低左右雙列表的從屬混淆 以表格作為總覽，Drawer 作為單一群組的詳情與編輯區。

整體觀察

用較短篇幅說明這次調整的共同方向

調整方向 原因 目前做法
降低工作流式操作門檻 原工作流模組較偏技術設定，對一般管理者不夠友善 改用固定線性流程、節點內直接設定、減少工具箱概念
補齊實際企業簽核情境 審核對象不只固定群組，也包含直屬主管、職務代理人等動態角色 在流程中支援動態帶入，並反推帳號管理需補設定欄位
減少設定來回跳轉 若流程建立完還要再回表格套用，容易漏設定也增加操作成本 在流程頁與表格頁都補上直接套用與查看套用位置的入口
保留後續擴充彈性 未來節點可能仍需與工作流共用，但當前審核情境不必做到同等複雜 先把審核流程收斂成簡化版本，之後再視需求擴張
讓群組標籤成為共用基礎資料 群組標籤未來不只服務簽核，也可能用於權限、通知與工作流 主位置放在系統管理，流程設定中只作為選用入口
帳號管理與群組標籤已整合於同一個系統管理原型，後續可再串接實際組織資料與權限來源。
