# 02 功能規格書 (FSD)

## 1. 資料修改與刪除機制 (CRUD Logic)
> 回應提問 1：如何修正匯入錯誤之數據。

### 1.1 修改機制
- 系統提供 RESTful PUT 接口針對單筆 `sample_id` 進行數據修正。
- 欄位修正後，系統將自動觸發重新計算 (Re-calculation) 該計畫之統計值與管制圖界限。
### 1.2 刪除機制
- 系統提供物理刪除 (Hard Delete) 接口，可刪除過時或錯誤計畫。

## 2. 數據重複性處理規則 (Deduplication)
> 回應提問 2：相同 管制項目+料號+批號 之處理。

### 2.1 判定準則 (Business Key)
系統以以下欄位之組合判定為一組「量測批次」：
- `ccm_name` (計畫名稱，由層別資訊組合)
- `characteristic_name` (管制項目)
- `batch_number` (批號)
- `created_at` (時間戳)

### 2.2 處置邏輯
- **預設行為**：追加 (Append)。
- **重複上傳偵測**：若偵測到 payload 之 `ref_id` 已存在，系統將報錯並拒絕寫入。

## 3. 計畫管理與變更彈性
> 回應提問 3：樣本數 n 變更規則。

### 3.1 樣本數固定原則
- 每一計畫下之樣本數 (n) 於建立時即鎖定（如 n=3）。
- **變更流程**：若製程抽樣標準由 n=3 改為 n=5，建議建立新的「版本分組」或新的計畫，以避免統計公式 (d2, c4) 之混淆。
